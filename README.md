# Tyler Sims Portfolio

Single-page portfolio site for Tyler Sims, focused on forecasting, dashboards, workflow automation, reporting systems, and decision-support tooling.

## Local use

Open [index.html](./index.html) in a browser, or serve the folder with any static file server.

## Before publishing

Update these placeholders:

- Update the GitHub profile URL in [index.html](./index.html) if it changes.
- Update the resume URL in [index.html](./index.html) if the hosted resume link changes.
- Update contact details if a non-work email should be used publicly.

## GitHub Pages deployment

This site is set up to work as a static GitHub Pages site.

1. Create or rename the repository to `<username>.github.io` if this will be a user site.
2. Push the portfolio files to the default branch.
3. In GitHub, open `Settings > Pages`.
4. Under `Build and deployment`, choose `Deploy from a branch`.
5. Select the branch to publish from, typically `main`, and choose `/ (root)` as the folder.
6. Save the settings and wait for GitHub Pages to publish the site.

If you prefer a project site instead of a user site, GitHub Pages can also publish from a standard repository and branch/folder source.
